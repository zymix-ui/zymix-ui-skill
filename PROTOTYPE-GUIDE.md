# ZymixUI 原型制作指南(设计师 / 产品)

无需绘图或编码。安装技能后,用一句话描述需求,AI 即可产出一张**符合 ZYMIX 设计规范**的高保真手机页面。
工程师获取设计变量(Tokens)请见 [TOKENS-GUIDE.md](./TOKENS-GUIDE.md)。

---

## 三步上手

1. **安装**:将 `zymix-ui-prototype.skill` 发送给 Claude,点击卡片上的 **Save skill**。
2. **描述需求**:例如「用 ZYMIX 做一个登录页」。
3. **获得结果**:AI 产出可在手机 / 浏览器直接打开的 HTML 页面,配色符合规范、深浅色跟随系统、并已通过合规检查。

规范细节(色值、圆角、字号、图标)均由技能负责。**只需描述"要什么内容",无需描述"样式怎么做"。**

---

## 如何描述需求

### 1. 说明页面与内容,不必给色值
- 推荐:「做一个 ZYMIX 转账确认页:收款人卡片、金额输入、底部主按钮 Confirm」。
- 不推荐:「主色 #26D93E,圆角 22…」——指定色值可能与规范冲突,配色交由技能处理。

### 2. 用设计系统词汇点名组件
AI 会据此精确调用对应组件:

| 你的说法 | 对应组件 |
|---|---|
| 主按钮 / 次按钮 / 危险按钮 / 文字按钮 | btn-primary / btn-secondary / btn-danger / btn-glass |
| 好友气泡 / 陌生人气泡 | bubble-friend / bubble-stranger |
| 品牌页头(大标题) / 导航页头(居中标题 + 返回) | header-brand / header-nav |
| 玻璃底部导航 / 玻璃圆钮 | tabbar / btn-glass-42 |
| Toast 提示 | 顶部点击触发的 toast |
| 列表 / 卡片 | list-row / surface 卡片 |

### 3. 深浅色自动跟随系统
生成的页面只有一份,切换系统外观即随之变化。如需锁定,可注明「锁定浅色 / 深色」。

### 4. 迭代使用规范语言
例如:「次要文字改为 muted」「这条分隔线用强分隔」「按钮降级为 secondary」「间距按设计稿取 8px」。

### 5. 提供设计稿即 1:1 还原
附上 Figma 截图或链接并注明「按此 1:1 还原」,技能会精确复刻每处细节(不对称圆角、描边、缩进、图标)。若仅作参考,可注明「参考该风格自由发挥」。

### 6. 支持 PRD 与外部原型
可提供 PRD 文档,或由其他工具生成的(不合规)原型。AI 只提取**需求意图**,以 ZymixUI 规范重做,不照搬不合规的样式。涉及多屏的 PRD,会先拆出页面清单并确认制作顺序。

### 7. 信息不全时会先确认
无论提供图片还是文字,AI 都会先分析布局与实现方案:信息充分则复述方案供确认,信息不足则先提出若干问题。因此简要描述即可,AI 会补全缺口。

### 8. 样式锁定、布局自由
布局可自由提出;若要求偏离规范的样式(如「标题用 22px」),技能会将其归入最接近的规范档位(20 或 24)——此为设计特性,用于保证一致性。

## 输出约定(技能默认遵循)

- **单页面**:一个 HTML 对应一台手机,手机上打开即全屏。
- **全英文文案**:输出页面文案统一为英文(面向英国市场)。
- **图片占位**:配图使用免版权图 + 骨架底色,图片加载失败时仍显示占位色块,不留空白。
- **图标来自图标库**:所有图标取自 ZymixUI 图标库,不手绘。
- **多状态以「灵动岛」演示**:页面若含加载 / 空 / 失败等状态,顶部会提供可展开的「灵动岛」用于原地切换预览,并内置系统 / 浅色 / 深色外观切换。

## 示例 Prompt(可直接改写使用)

**基础**
> 做一个 ZYMIX 空状态页:居中插画占位、一句 "No messages yet",下方主按钮 "Start chatting"。

**含组件点名**
> 做 ZYMIX 删好友确认页:导航页头 Remove friend + 玻璃返回钮,中部好友头像与说明,点击主按钮弹出 Toast "This can't be undone",底部 dangerSoft 按钮 Remove + ghost 按钮 Cancel。

**列表页**
> 做 ZYMIX 消息列表页:品牌页头 + 搜索框,8 条会话(头像、昵称、末条消息 muted、时间、未读徽标),底部玻璃导航。

**1:1 还原**
> [附 Figma 截图] 按此 1:1 还原,间距与圆角需对齐,完成后对照设计稿自检。

**IM 聊天**
> 做 ZYMIX 与好友 Hannah 的聊天页:导航页头带头像,五六条来回消息使用好友气泡,末条发出消息显示已读回执,底部玻璃输入条。

## 验收要点

- 每个页面均会运行合规检查(`check_compliance.py`),可要求 AI 附上检查结果。
- 交互(Toast、按钮按压、灵动岛切换)需在浏览器或真机验证,预览卡片不执行 JS。
- 深浅色:切换系统外观,或点击灵动岛的外观图标查看效果。
