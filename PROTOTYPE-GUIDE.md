# 做原型指南(设计师 / 产品)

> 不用会画图、不用写代码。装上技能，说一句话，AI 就给你一张**完全符合 ZYMIX 规范**的高保真手机页面。
> 工程师取用 tokens 请看 [TOKENS-GUIDE.md](./TOKENS-GUIDE.md)。

---

## 三步上手

1. **装技能**：把 `zymix-ui-prototype.skill` 发给 Claude，点卡片上的 **Save skill**。
2. **说需求**：`用 ZYMIX 做一个登录页`。
3. **拿结果**：AI 产出一个手机/浏览器直接打开的 HTML 页面——真实规范配色、深浅色跟随系统、跑过合规检查。

就这么简单。规范细节（颜色值、圆角、字号、图标）全是技能的事，**你只描述"要什么"，不描述"怎么做样式"**。

---

## 怎么下需求(说清"要什么")

### ① 描述页面 + 内容,别喂色值
- ✅ 「做一个 ZYMIX 转账确认页:收款人卡片、金额输入、底部主按钮 Confirm」
- ❌ 「主色 #26D93E,圆角 22...」—— 喂色值反而可能污染规范,技能自己知道。

### ② 用设计系统词汇点名组件(AI 会精确调用)

| 你说 | AI 调用 |
|---|---|
| 主按钮 / 次按钮 / 危险按钮 / 文字按钮 | btn-primary / btn-secondary / btn-danger / btn-glass |
| 好友气泡 / 陌生人气泡 | bubble-friend / bubble-stranger |
| 品牌页头(大标题) / 导航页头(居中标题+返回) | header-brand / header-nav |
| 玻璃底部导航 / 玻璃圆钮 | tabbar / btn-glass-42 |
| Toast 提示 | 顶部点击触发的 toast |
| 列表 / 卡片 | list-row / surface 卡片 |

### ③ 深浅色自动跟随系统
生成的页面只有一份,切系统外观就跟着变。要锁定某模式才说「锁定浅色 / 深色」。

### ④ 迭代用规范语言
「次要文字改成 muted」「这条线用强分隔」「按钮降级成 secondary」「间距按设计稿 8px」。

### ⑤ 有设计稿 = 1:1 还原
贴 Figma 截图或链接,明说「按这张 1:1 还原」——技能会精确复刻每个细节(不对称圆角、描边、缩进、图标)。只找灵感就说「参考这个风格自由发挥」。

### ⑥ PRD / 别的 AI 生成的原型也能喂
把 PRD 文档、或别处生成的(不合规)原型丢给 AI——它只提取**需求意图**,用 ZymixUI 规范重做,不照搬不合规样式。多屏 PRD 会先拆出页面清单跟你确认先做哪几屏。

### ⑦ 信息不全 AI 会先问你
给图或文字,AI 都会先分析布局+方案:信息够就复述方案让你确认,不够就问几个问题理清再生成。放心简略描述,AI 会补齐。

### ⑧ 样式锁死、布局自由
布局随便提;要求偏离规范的样式(如"标题 22px")会被拒并归到最近规范档(20 或 24)——这是特性,保证一致性。

## 输出约定(技能默认帮你做到)

- **单页面**:一个 HTML 一台手机,手机上打开即全屏。
- **全英文**:输出页面文案一律英文(面向英国市场)。
- **真图占位**:配图用免版权图 + 骨底色块,图挂了也有占位、不空白。
- **只用图标库**:图标全部来自 ZymixUI 图标库,绝不手绘。
- **多状态用"灵动岛"演示**:一个页面若有 加载/空/失败 等状态,顶部会有个可点开的"灵动岛"原地切换预览,并自带 系统/浅/深 外观切换。

**带组件点名**

**1:1 还原**
> [贴 Figma 截图] 按这张 1:1 还原,间距圆角都要对上,做完对比设计稿自检。

## 验收技巧
- 每页 AI 都会跑合规检查(`check_compliance.py`),可直接问「贴一下合规检查结果」。
- **交互(Toast、按钮按压、灵动岛切换)要在浏览器或真机验证**,预览卡片不跑 JS。
- 深浅色:切系统外观、或点灵动岛的外观图标看效果。
