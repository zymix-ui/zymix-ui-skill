# ZymixUI 原型技能 · 使用说明

把 ZymixUI 设计系统(颜色/间距/字号 token + Button/Toast/玻璃材质组件 + 3 个页面模板)打包成的 AI 技能。装上后让 AI 画 ZYMIX 界面,自动按规范出图:正确品牌绿、0.5 发丝线、按钮配色配对、液态玻璃、深浅色跟随系统。**单页面输出**,手机上打开即全屏预览。

---

## 安装
聊天里点 `.skill` 文件卡片上的「Save skill」。之后说"画一个 ZYMIX 的 XX 页面"就自动触发。

---

## Prompt 核心原则:说清"要什么",不说"怎么做样式"

规范细节(颜色值、圆角、字号)是技能的事,你只需描述**页面类型 + 内容 + 用哪些组件**。

### ① 描述页面和内容,别喂色值
- ✅ 「做一个 ZYMIX 转账确认页:收款人卡片、金额输入、底部主按钮 Confirm」
- ❌ 「做个页面,主色 #26D93E,圆角 22...」——喂色值反而可能污染规范,技能自己知道

### ② 用设计系统词汇点名组件(AI 会精确调用)
| 你说 | AI 调用 |
|---|---|
| 主按钮 / 次按钮 / 危险按钮 / 文字按钮 | btn-primary / btn-secondary / btn-danger / btn-glass |
| 好友气泡 / 陌生人气泡 | bubble-friend / bubble-stranger |
| 品牌页头(大标题) / 导航页头(居中标题+返回) | header-brand / header-nav |
| 玻璃导航 / 玻璃圆钮 | nav-bar / btn-glass-42 |
| Toast 提示 | 顶部点击触发的 toast |
| 列表 / 卡片 | list-row / surface 卡片 |

### ③ 深浅色自动跟随系统
生成的页面只有一份,切系统外观就跟着变。要锁定某模式才说「锁定浅色」。

### ④ 迭代时用规范语言
「次要文字改成 muted」「这条分隔线用强分隔」「按钮降级成 secondary」「间距按设计稿 8px」。

### ⑤ PRD / 非规范原型也能喂
产品经理给的 PRD 文档、或用别的 AI 生成的(不合规的)原型,直接丢给 AI——它会**只提取需求意图,用 ZymixUI 规范重做**,不会照搬那些不合规的样式。PRD 涉及多屏时,AI 会先拆出页面清单跟你确认先做哪几屏,再逐页生成。

### ⑥ 信息不全时 AI 会先问你
不管给参考图还是文字描述,AI 会先分析布局+实现方案;信息够就复述方案让你确认,不够就问你几个问题理清再生成——所以放心大胆地简略描述,AI 会补齐缺口。

### ⑦ 有设计稿 = 要 1:1 还原
贴 Figma 截图或链接时,明说「按这张 1:1 还原」——技能会精确复刻每个细节(不对称圆角、描边、缩进、图标)。只是找灵感就说「参考这个风格自由发挥」。

### ⑧ 样式锁死、布局自由
布局(结构、内容组织)随便提;若你要求偏离规范的样式(如"标题用 22px"),技能会拒绝并归到最近规范档(20 或 24)——这是特性,保证一致性。

## 案例演示(照抄改词即可)

**案例 1 · 最简单**
> 做一个 ZYMIX 的空状态页:居中一个插画占位、一句"还没有消息"、下面一个主按钮"开始聊天"。

**案例 2 · 带组件点名**
> 做 ZYMIX 删好友确认页:导航页头标题 Remove friend + 玻璃返回钮,中间好友头像和说明,点主按钮弹出 Toast 提示"This can't be undone",底部 dangerSoft 按钮 Remove + ghost 按钮 Cancel。

**案例 3 · 列表页**
> 做 ZYMIX 消息列表页:品牌页头 ZYMIX + 搜索框,下面 8 条会话(头像、昵称、最后一条消息 muted、时间、未读徽标),底部玻璃导航三个 tab。

**案例 4 · 1:1 还原设计稿**
> [贴 Figma 截图] 按这张 1:1 还原,间距和圆角都要对上,做完对比设计稿自检。

**案例 5 · IM 聊天**
> 做 ZYMIX 和好友 Hannah 的聊天页:导航页头带头像,五六条来回消息用好友气泡,最后一条我发的显示已读回执,底部玻璃输入条。

## 验收技巧
- 每个页面 AI 都会跑 `check_compliance.py` 合规检查,可直接问「贴一下合规检查结果」
- **交互(Toast/按钮按压)必须在浏览器或真机验证**,预览卡片不跑 JS
- 深浅色:手机/浏览器切系统外观看跟随效果
