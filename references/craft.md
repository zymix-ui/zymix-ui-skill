# ZYMIX craft.md — 工艺层(品味声明)

> 把"高级 / 克制"这类抽象审美词,翻译成 Agent 能直接执行的规则。方法:每条 = 可执行规则 + 反面(anti-pattern),让品味从"看感觉"变成"可检查"。
> **分工**:`spec` 管功能边界,`domain` 管业务语义,`design.md`(=tokens/规范)管视觉事实源,`components` 管组件语义,`template` 管结构骨架,**`craft.md`(本文件)管"工艺质量"——挡住 AI 味,守住 ZYMIX 的调性**。
> 合规检查(check_compliance.py)只查"对不对";craft 查"平庸不平庸"。

## 0. ZYMIX 调性锚点(三个形容词)

**克制 · 高级 · 有呼吸感**。绿+白的 iOS 社交系统,靠留白和层级说话,不靠色块和特效。拿不准时,永远选更素、更空的那一版。

## 1. 反 AI 味清单(最高优先级,命中即打回)

分布收敛会把模型拽向训练集里最高频、最安全的那类结果。以下是 ZYMIX 里必须挡住的套路:

- **禁渐变铺底**:不用 linear/radial 渐变当背景或图片位。ZYMIX 是纯色 token 系统。反面:蓝紫渐变 hero、彩色渐变卡。(唯一例外:设计系统自带的 `.nav-fade` 渐隐遮罩。)
- **禁彩虹圆圈墙**:分类/入口不要一排高饱和撞色圆。反面:红橙黄绿蓝各一个的分类圈——廉价、吵。做法:中性圆底(surface-secondary)+ 细线单色图标。
- **禁等大白卡墙**:不要把内容全塞进一排大小相同的卡片。用留白、分组、主次尺寸区分层级。
- **禁大写标签当装饰**:不用 ALL-CAPS 英文小标签堆气氛。中文产品说人话。
- **禁彩色块堆信息**:日期/名额/天气等信息不要各占一个撞色小卡。用文字层级 + 留白表达。
- **禁描边+阴影堆砌**:ZYMIX 阴影体系已废弃,靠 surface 层级色区分,不加投影。

## 2. 用色工艺:accent 预算制

- **accent(品牌绿)是稀缺资源**,整屏只给"最想让用户点的那一个动作"+ 当前导航选中态。一屏 accent 实心块**不超过 2 处**。
- 反面:主按钮绿、信息卡绿、FAB 绿、链接绿、徽标绿全上——绿到失去焦点,等于没有重点。
- 其余一切用中性层级(foreground base/muted/subtle + surface 层级)表达。
- 状态色(danger/info)只在真正表达该语义时出现,不做装饰色。

## 3. 留白与密度

- 首屏信息**做减法**:一个清晰的主推 + 一组次要列表,足矣。不要把六种模块塞满一屏。
- 区块间距 ≥ 20px,页边距 16px 起。宁可让用户滚动,也不要挤。
- 反面:上来就搜索+分类+推荐+榜单+活动流五段并列。

## 4. 图像策略(照片位)

- 活动/内容缩略优先**真实照片位**:中性底(surface-secondary)+ 低透明度细线插画示意,明确是"待填照片"的占位,而非假装成图的高饱和色块。
- 反面:用高饱和纯色/渐变冒充照片——显廉价。
- 真上线时接真实图片;原型阶段用中性占位,不用彩色填充。

## 5. 图标工艺

- 线性图标,stroke 统一 **1.5px**,currentColor 继承文字色,风格一致(圆头圆角)。
- 反面:粗细不一、实心线性混用、给图标单独上撞色。

## 6. 文字层级

- 层级靠**字号 + 字重**拉开(t-title / t-label / t-body 角色),不靠颜色。
- 次要信息用 `--foreground-muted`,辅助信息才用 `--foreground-subtle`;不要用 subtle 表达正文。
- 反面:用不同颜色区分同级文字。

## 8. 灵动岛约定(默认)

灵动岛**按页面角色分两型**,都黑底白字、`position:absolute` 锚在状态栏中间、不占布局、收起态高 32 与系统状态栏同高:

**A 型 · 状态预览岛**——页面有多状态时(加载 / 空 / 失败 / 权限 / 成功,见 `spec.md`),**默认用它在同一页原地演示**,不平铺多个页面框:
- 收起药丸宽约 104;点药丸 → 平滑形变**展开**(宽度填满、左右各留 16px,圆角 `--radius-5xl` 32px,`height` 过渡,**先展形、选项延后淡入**,`cubic-bezier(.32,.72,0,1)` ~0.4s)。
- 展开菜单 flex-wrap 放状态胶囊,点选项原地切状态;点药丸或**点岛外区域**收起。
- 外观切换见下(A 型里做成叠在状态标签右侧的单图标)。

**B 型 · 外观岛**——无多状态的静态页(me / chat / 资料页等):不需要展开,**药丸本身就是外观开关**:`[圆点 · 模式名 · 图标]`,**点一下循环** 系统 / 浅 / 深,无菜单、无展开。

**外观切换(两型通用)= 单个图标点击循环**,图标取自图标库(**勿手绘**;库里没有从全量库 `zymix-icons/svgs` 取并补进 `icons-bundled.json`):系统=`display`、浅色=`sun`、深色=`moon`。整屏换肤即反馈,不放文字说明。A 型放状态标签右侧绝对定位(`left:calc(50% + 30px)`,不挤动标签,展开才显);B 型直接在药丸内。
- 实现:JS 强制浅=同时 `root.classList.add('lite')` + `setAttribute('data-theme','light')`,强制深=`setAttribute('data-theme','dark')`,系统=两者都清空;CSS 用 `.lite` / `:root:not(.lite)` 或 `:root:not([data-theme="light"])` 门控系统深色 + `[data-theme="dark"]` 强制深。**静态 HTML 里绝不写死 data-theme 属性**(只在 CSS 选择器和运行时 JS 用),否则合规判双模式 FAIL。图标随 `data-mode` 用 `.ic-*{display:block}` 切换。
- 岛始终黑底白字(不随页面主题翻转);白色半透明(分隔线等)用 `var(--default-white)` + `opacity`,勿写裸 rgba;岛内按钮加 `-webkit-tap-highlight-color:transparent` 防移动端点击闪块。
- 演示脚手架真实交付时移除(外观开关可保留,便于走查两套色)。

**例外**:仅当用户明确要求"铺开 / 并排多个页面"时,才平铺多个页面框。
