# ZYMIX spec.md — 功能状态声明

> **spec**:在 ZYMIX 设计体系里,管**功能边界与状态流**的一层——一个页面在"主流程能跑"之外,还必须定义空态、加载、失败、权限、成功反馈等所有非理想状态。这与尼尔森可用性原则中的"系统状态可见""帮助用户识别与恢复错误""防错"一脉相承。
> **分工**:`spec`(本文)管"功能是否成立";`design.md`(tokens/规范)管视觉事实源;`components` 管组件语义;`template` 管结构骨架;`craft.md` 管工艺/调性。
> **判定原则**:主流程能跑 ≠ 页面成立。缺状态 = 功能没做完,不是"待优化"。
> **用法**:生成页面前,先按"通用清单 + 对应页型矩阵"确认要覆盖哪些状态;evaluator 的"交互就绪"维度逐条核,缺失回流到本文件。

---

## 一、通用状态清单(所有页面必须逐条回答)

每个状态标注:**定义 → 绑定实现(token/组件)→ 是否阻断**。阻断项缺失即不可交付。

1. **加载中 / 骨架**
   有网络请求的区域首帧必须有骨架,不留白屏。绑定:`--skeleton-base` 底 + `--skeleton-highlight` 微光;形状复刻真实内容轮廓(头像圆、文字条)。**阻断:是**(首屏白屏直接打回)。

2. **空态(无数据)**
   请求成功但无内容。绑定:居中 zymix 图标(`--foreground-subtle`)+ 标题 `.t-label-base` + 说明 `.t-body-sm`(`--foreground-muted`)+ 一个引导下一步的主按钮 `.btn-primary`。文案说清"为什么空 + 下一步做什么",禁止只写"暂无数据"。**阻断:是**。

3. **加载失败 / 错误**
   请求失败。绑定:说明文字用 `--danger-soft-foreground`(不是大面积红块)+ **重试按钮** `.btn-outline`/`.btn-tertiary`。必须给出可恢复路径。**阻断:是**(无重试入口直接打回)。

4. **无权限 / 未登录**
   无访问权或未登录。绑定:说明 + 授权/登录 CTA `.btn-primary`;不能静默空白或直接报错。**阻断:是**(涉及登录/隐私的页面)。

5. **成功反馈**
   写操作(报名/发送/保存)完成后必须有明确反馈。绑定:`.toast.glass`(中性图标,不用状态色)或就地状态切换(如按钮→已报名)。**阻断:是**(写操作无反馈直接打回)。

6. **提交中 / 防重复**
   写操作进行中:触发按钮进入 loading + `disabled`(整体 `opacity:.5`),防重复提交。**阻断:是**(可重复提交的写操作)。

7. **弱网 / 离线**
   网络异常。绑定:顶部提示条或 `.toast.glass`;进行中的操作进入"排队/待发送",不丢失。**阻断:否**(建议覆盖)。

8. **边界内容**
   超长文本(截断 + 省略号)、超大数字(1,208→缩写规则)、缺省头像(`--skeleton-base` 兜底)、单行/极多行。**阻断:否**(建议覆盖)。

9. **分页 / 加载更多**
   列表滚到底:底部 loading → "没有更多了"(`.t-body-xs` + `--foreground-subtle`)。**阻断:否**。

## 二、页型状态矩阵

每个页型在通用清单之上,额外必须覆盖以下**页型专属状态**。

### A. 动态流 / 列表(feed、活动列表、会话列表)

| 状态 | 绑定实现 | 阻断 |
| --- | --- | --- |
| 首屏骨架 | 列表行骨架 ×5~8,`--skeleton-base` | 是 |
| 空态 | 居中图标+文案+主 CTA(如"发起活动") | 是 |
| 加载失败 | 整页错误 + 重试 `.btn-outline` | 是 |
| 下拉刷新 | 顶部刷新指示 | 否 |
| 上拉加载更多 / 到底 | 底部 loading → "没有更多了" | 否 |
| 单条删除后变空 | 回落到空态(不是留白) | 是 |
| 会话列表:置顶行 | 背景 `--surface-secondary` | 否 |
| 会话列表:未读徽标 | `--accent-base` 圆 + 白字 11px | 否 |

### B. 聊天会话页(IM)

| 状态 | 绑定实现 | 阻断 |
| --- | --- | --- |
| 历史消息加载(顶部) | 顶部 loading,上拉加载更早 | 否 |
| 发送中 | 气泡带发送中指示 | 是 |
| **发送失败 / 可重发** | 气泡旁红色感叹号(`--danger-soft-foreground`)+ 点击重发 | 是 |
| 已读回执 | 好友 `--feature-im-read-friend`,陌生人 `--feature-im-read-stranger` | 否 |
| 对方输入中 | "对方正在输入…"提示 | 否 |
| 空会话(新好友) | 居中引导语,鼓励发第一条 | 否 |
| 不可发送态(拉黑/仅好友) | 输入条禁用 + 说明 | 是 |
| 无网络 | 发送排队,不丢消息 | 是 |

> 好友气泡 `.bubble-friend`,陌生人 `.bubble-stranger`;气泡圆角 `--radius-2xl`。

### C. 表单 / 详情页(报名、资料编辑、活动详情)

| 状态 | 绑定实现 | 阻断 |
| --- | --- | --- |
| 详情加载骨架 | 大图+标题+正文骨架 | 是 |
| 详情加载失败 | 错误 + 重试 | 是 |
| 字段校验错误 | 内联错误文字 `--danger-soft-foreground` + 图标,不只靠红框(色盲不可见) | 是 |
| 提交中 | 主按钮 loading + `disabled` | 是 |
| 提交成功 | `.toast.glass` 或跳转确认页 | 是 |
| 提交失败 | 说明 + 重试,已填内容不清空 | 是 |
| 名额已满 / 已报名 | 主按钮禁用态 + 说明("已满"/"已报名") | 是 |
| 未登录拦截 | 拦截到登录 CTA | 是 |
| 离开未保存 | 二次确认(`--backdrop-base` + 面板) | 否 |

### D. 个人主页(Me / 他人主页)

| 状态 | 绑定实现 | 阻断 |
| --- | --- | --- |
| 资料加载骨架 | 头像+昵称+统计骨架 | 是 |
| 加载失败 | 错误 + 重试 | 是 |
| 自己 vs 他人视角 | 自己="编辑资料";他人=关系操作 | 是 |
| 关系态 | 加好友 `.btn-primary` / 已请求(禁用)/ 好友(次要按钮) | 是 |
| 隐私(仅好友可见) | 受限区显示"仅好友可见"占位,不报错 | 是 |
| 拉黑 / 被拉黑 | 对应受限说明 | 是 |
| 内容为空(还没动态) | 分区空态,不留白 | 否 |

## 三、阻断闸门(任一命中即不可交付)

- 有请求的首屏**无骨架**(白屏)
- 列表/内容**无空态**,或空态只写"暂无数据"、无下一步
- 加载失败**无重试入口**
- 写操作(报名/发送/保存)**无成功反馈**,或**可重复提交**
- 表单**无字段校验错误态**
- 涉及登录/隐私的页面**无未登录/无权限态**
- IM **发送失败不可重发**

## 四、evaluator 回流

evaluator 的**「交互就绪」**维度按本文件逐条核。发现问题时,报告里写明"缺哪个状态 → 回流 spec.md",并判断是否为阻断项。示例:

- 截图/点击发现"报名后无反馈" → 交互就绪扣分,**阻断**,回流 spec.md §一.5
- "列表空但只显示白屏" → **阻断**,回流 spec.md §二.A 空态
- "无网络时消息静默丢失" → **阻断**,回流 spec.md §二.B 无网络

> 与其他声明的边界:状态"长什么样"(色/字/间距)归 `design.md`,"该不该有这个状态"归本文件。两者都缺时,先补 spec(有没有),再补 design(好不好看)。
